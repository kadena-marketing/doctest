# Kadena Command-Line Reference

The **Kadena command-line interface** (`kadena-cli`) provides direct access to the Kadena blockchain and commands to create, test, deploy, and manage applications for the Kadena network. You can use the Kadena command-line interface interactively or in scripts and automated workflows.

## Prerequisites

Before using the Kadena command-line interface, verify the following basic requirements:

<Tabs>
  <TabItem value="node" label="Node.js" default>
    Install Node.js version 18 or later from the [official Node.js website](https://nodejs.org/).
  </TabItem>
  <TabItem value="pnpm" label="pnpm">
    Install the pnpm package manager. You can install it using npm:

```bash
npm install -g pnpm
```

Verify pnpm is install by running the following command:

```bash
pnpm --version
```

## Step 1: Installation - Kadena CLI

The Kadena CLI is packaged in a TypeScript library. You can install the Kadena CLI (`@kadena/kadena-cli`) using the npm or pnpm package manager. Choose your preferred package manager:

<Tabs>
<TabItem value="npm" label="npm" default>
     Install globally using npm:

     ```bash
     npm install -g @kadena/kadena-cli
     ```
</TabItem>
<TabItem value="pnpm" label="pnpm">
     Install globally using pnpm:

     ```bash
     pnpm install -g @kadena/kadena-cli
     ```
</TabItem>
</Tabs>

### Verify Installation

To verify the package is installed and display usage information, run:

```bash
kadena
```

## Step 2: Command Overview

You can use the `kadena` parent command with different flags and subcommands to perform various operations. The basic syntax is:

```bash
kadena <subcommand> <action> [arguments] [flags]
```

<Tabs>
  <TabItem value="Overview" label="Overview" default className={styles.customTabItem}>

The Kadena CLI is organized into categories (subjects) that describe the action you want to perform. Here's a quick overview:

```plaintext
kadena <subject> [...<subject>] <action> [--flags] [args]
```

For example, to manage wallet-related information:

```bash
kadena wallet <action>
```

  </TabItem>

  <TabItem value="Subjects" label="Available Subjects" className={styles.customTabItem}>

### Available Subjects

| Command Subject | Description |
|-----------------|-------------|
| **`config`**        | Configure the CLI. |
| **`dapp`**          | Create and manage applications. |
| **`wallet`**        | Manage wallets and keys. |
| **`key`**           | Manage keys. |
| **`account`**       | Manage accounts. |
| **`network`**       | Manage network information. |
| **`tx`**            | Manage transactions. |
| **`help`**          | Display help. |
| **`version`**       | Display version information. |

  </TabItem>
</Tabs>

### Get Command-Specific Help

To get help on a specific subject, use the `--help` flag:

```bash
kadena <subject> --help
```

### Use Interactive Execution

The Kadena CLI supports **interactive execution**. If you run a command without all required options, the CLI will prompt you for the missing information.

### Example: Adding a New Wallet

To add a new wallet interactively:

```bash
kadena wallet add
```

The CLI will guide you through the necessary steps.


### Global Flags

Global flags can modify the behavior of commands across the Kadena CLI:

| Flag        | Description                                                      |
|-------------|------------------------------------------------------------------|
| **`--help`**  | Display usage information.                                      |
| **`--version`** | Display version information.                                 |
| **`--quiet`**   | Eliminate interactive prompts and confirmations.             |
| **`--json`**    | Format output in JSON.                                       |
| **`--yaml`**    | Format output in YAML.                                       |

## Step 3: Use JSON or YAML Output

You can format output from commands in **JSON** or **YAML** by using the `--json` or `--yaml` flag.

## Step 4: Use Legacy Mode

Use the `--legacy` flag to maintain compatibility with earlier cryptographic standards:

```bash
kadena [command] --legacy
```

Legacy mode is available for commands such as:

- `kadena wallet add`
- `kadena key generate`
- `kadena tx sign`


## Troubleshooting

If you encounter issues during installation or usage:

### Check Dependencies


Ensure Node.js and pnpm are properly installed. You can check their versions:

```bash
node --version
pnpm --version
```

### CLI Issues

If you're having trouble with specific CLI commands, use the `--help` flag for detailed usage information:

```bash
kadena <command> --help
```

### Consult GitHub Issues

For common problems and community solutions, visit the [Kadena CLI GitHub Issues](https://github.com/kadena-io/kadena-cli/issues) page.
